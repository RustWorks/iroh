State of the plumtree.

Fields§

Our address.

§config: Config

Configuration for this plumtree.

§eager_push_peers: HashSet<PI>

Set of peers used for payload exchange.

§lazy_push_peers: HashSet<PI>

Set of peers used for control message exchange.

§lazy_push_queue: HashMap<PI, Vec<IHave>>§missing_messages: HashMap<MessageId, VecDeque<(PI, Round)>>

Messages for which a MessageId has been seen via a Message::IHave but we have not yet received the full payload. For each, we store the peers that have claimed to have this message.

§received_messages: TimeBoundCache<MessageId, ()>

Messages for which the full payload has been seen.

§cache: TimeBoundCache<MessageId, Gossip>

Payloads of received messages.

§graft_timer_scheduled: HashSet<MessageId>

Message ids for which a Timer::SendGraft has been scheduled.

§dispatch_timer_scheduled: bool

Whether a Timer::DispatchLazyPush has been scheduled.

§init: bool

Set to false after the first message is received. Used for initial timer scheduling.

§stats: Stats

Stats of this plumtree.

fn on_ihave(&mut self, sender: PI, ihaves: Vec<IHave>, io: &mut impl IO<PI>)

Handle receiving a Message::IHave.

When a node receives a IHAVE message, it simply marks the corresponding message as missing It then starts a timer, with a predefined timeout value, and waits for the missing message to be received via eager push before the timer expires. The timeout value is a protocol parameter that should be configured considering the diameter of the overlay and a target maximum recovery latency, defined by the application requirements. This is a parameter that should be statically configured at deployment time. (p8)
